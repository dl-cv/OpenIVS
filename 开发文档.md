# 开发文档

## 文档规则

- 文档只记录项目当前实现与运行事实。
- 文档不写建议、猜测、规划、风险判断、设计理由、判定依据。
- 文档不写问答式内容，不直接回答对话中的临时问题。
- 文档不写历史、过程、修改记录、拆分说明。
- 文档不写重复内容；同一事实只在最合适的位置出现一次。
- 文档不写第一人称、第二人称对话。
- 文档规则只保留在本文件，子文档不重复书写规则。
- 文档正文只描述项目本身，不写对话提示词、写作要求和编写过程。
- 目标：任何开发者仅依据本文档，都能实现一个**功能与用户体验一致**的程序。允许代码结构不同，但最终呈现给用户的功能、交互、输出文本与可视化效果必须一致。

## 文档总览

- `开发文档.md`：开发文档索引、文档规则与统一编译说明。
- `C# API开发文档.md`：`DlcvCsharpApi` 的工程结构、依赖、运行模式、输出结构与 Flow 模块。
- `C#测试程序开发文档.md`：`DlcvDemo` 的界面、交互、推理流程与结果展示。
- `C#测试程序2开发文档.md`：`DlcvDemo2` 的三模型固定推理流程、界面与结果展示。
- `C#测试程序3开发文档.md`：`DlcvDemo3` 的两模型串联、固定裁图、batch 推理与速度测试。
- `C#模型测试工程开发文档.md`：`Test/DlcvCSharpTest` 与 `Test/dlcv_infer_cpp_test` 的控制台测试范围、输出与运行方式。
- `C++API开发文档.md`：`dlcv_infer_cpp_dll` 的构建配置、依赖、数据结构与 Flow 节点。
- `C++测试程序开发文档.md`：`dlcv_infer_cpp_qt_demo` 的 Qt 界面、依赖、运行方式与交互行为。
- `C++测试程序3开发文档.md`：`dlcv_infer_cpp_qt_demo3` 的两模型串联、固定裁图、batch 推理与界面规格。

## 工程关系

- `DlcvCsharpApi` 为 `DlcvDemo`、`DlcvDemo2`、`DlcvDemo3` 和 `Test/DlcvCSharpTest` 提供 C# 推理接口。
- `dlcv_infer_cpp_dll` 为 `dlcv_infer_cpp_qt_demo`、`dlcv_infer_cpp_qt_demo3` 和 `Test/dlcv_infer_cpp_test` 提供 C++ 推理接口。

## 统一编译说明

- 本仓库所有编译、构建、重建与发布前构建验证统一通过项目级 skill 脚本 `.cursor/skills/vs-build/scripts/build.py` 执行。
- 默认构建目标为仓库根目录 `OpenIVS.sln`。
- 指定单个 C# 项目时，构建对应 `.csproj`。
- 默认构建参数为 `Debug`、`x64`、`Build`、`minimal`。
- 用户明确指定 `Release`、`Rebuild`、`Clean` 等参数时，按指定值执行。
- 不直接使用 `msbuild`、`dotnet`、`powershell` 或其他本地 shell 编译命令。
- 构建目标、输出目录与运行入口仍以各子文档记录的工程文件、解决方案和产物为准。
- 本节为各子文档中编译说明的统一引用位置。
