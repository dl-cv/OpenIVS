# C++测试程序3开发文档

> 目标：定义 `dlcv_infer_cpp_qt_demo3` 的开发规格。程序业务逻辑固定为“两模型串联 + 原图固定裁图 + 模型2 batch 推理 + 结果统一回写原图坐标 + 只展示模型2结果”，开发与验收均以本文档为准。

## 1. 文档范围与定位

- 软件名称：`C++测试程序3`
- 项目目录：`dlcv_infer_cpp_qt_demo3`
- 工程文件：`dlcv_infer_cpp_qt_demo3/dlcv_infer_cpp_qt_demo3.vcxproj`
- 独立解决方案：`dlcv_infer_cpp_qt_demo3/build_only.sln`
- 技术栈：`C++17`、`Qt6 Widgets`、`OpenCV 4.10`、`nlohmann::json`、`dlcv_infer_cpp_dll`
- 启动入口：`dlcv_infer_cpp_qt_demo3/main.cpp`
- 核心界面与业务逻辑：`dlcv_infer_cpp_qt_demo3/MainWindow.h`、`dlcv_infer_cpp_qt_demo3/MainWindow.cpp`
- 图像显示控件：复用 `dlcv_infer_cpp_qt_demo/ImageViewerWidget.{h,cpp}`

本程序不是通用流程编辑器，也不是多模型路由平台。程序只围绕两个固定角色的模型工作：

- `模型1（定位）`：负责在整图上给出目标位置
- `模型2（识别）`：负责对每个目标的固定尺寸局部图执行 batch 推理

最终界面只展示模型2回写到整图坐标后的结果，不展示模型1中间结果。

### 1.1 程序概述

程序具备以下主体能力：

- Qt 主界面、日志区、图像显示区、进度条、状态文本已完成
- 支持分别选择模型1、模型2，并在选择完成后立即自动加载
- 支持选择单张图片，并在选择完成后立即自动触发一次推理
- 支持记忆模型1、模型2、图片最近一次使用路径
- 支持查看模型信息，并在模型加载成功后自动显示模型信息
- 支持固定裁图大小 `128 x 192`
- 支持模型2线程数设置，执行 batch 推理
- 支持速度测试
- 支持释放模型

### 1.2 关键交互要求

程序交互满足以下要求：

1. 用户浏览选择模型1或模型2后，应立即自动执行对应模型加载，不再要求先“浏览”再手动点“加载模型”才能生效。
2. 用户浏览选择图片后，应立即自动触发一次单次推理，不再要求额外点击“执行推理”才开始本次推理。
3. 主界面不再显示 `设备` 选择控件，也不再显示 `文档` 按钮。
4. 程序应记忆模型1、模型2、图片最近一次使用路径，并在下次启动或再次打开文件对话框时恢复这些路径。
5. 程序应提供 `获取模型信息` 入口，并在每次模型加载成功后自动显示已加载模型的信息。

说明：

- `执行推理` 按钮保留，用于对已选择图片重新执行推理。
- `加载模型1`、`加载模型2` 按钮可以保留；若保留，按钮行为与自动加载行为保持一致。

## 3. 功能定位

本程序是一个面向固定业务流程的 C++/Qt Demo，执行流程固定为：

1. 原图输入模型1
2. 模型1输出目标结果
3. 从每个目标提取中心点
4. 在原图上以中心点裁固定大小 `128 x 192` ROI
5. 模型2按 batch 推理各个 ROI
6. 模型2结果平移回写到整图坐标
7. 只显示模型2最终结果

程序不提供以下能力：

- 不提供可编辑的裁图宽高输入框
- 不提供 batch size 输入框
- 不提供通用流程编排
- 不提供文件夹批量推理
- 不提供模型路由、多分支流程编排
- 不提供模型1中间结果显示开关

## 6. 交互流程

### 6.1 浏览并自动加载模型1

- 点击 `浏览...` 选择模型1文件
- 路径写入 `模型1路径` 文本框
- 文件对话框默认定位到最近一次模型1路径所在目录
- 选择成功后，应立即自动执行与 `onLoadModel1()` 等价的加载逻辑
- 加载成功时，文本区输出“模型1加载成功”与完整路径
- 加载成功后，保存 `LastModel1Path`
- 加载成功后，自动显示模型信息
- 加载失败时，文本区输出异常信息，并弹出错误提示

### 6.2 浏览并自动加载模型2

- 点击 `浏览...` 选择模型2文件
- 路径写入 `模型2路径` 文本框
- 文件对话框默认定位到最近一次模型2路径所在目录
- 选择成功后，应立即自动执行与 `onLoadModel2()` 等价的加载逻辑
- 加载成功时，文本区输出“模型2加载成功”与完整路径
- 加载成功后，保存 `LastModel2Path`
- 加载成功后，自动显示模型信息
- 加载失败时，文本区输出异常信息，并弹出错误提示

### 6.3 选择图片并自动推理

- 点击 `浏览...` 选择待推理图片
- 路径写入 `图片路径` 文本框
- 文件对话框默认定位到最近一次图片路径所在目录
- 选择成功后，应立即自动触发一次与 `onInfer()` 等价的单次推理
- 执行推理前保存 `LastImagePath`
- 若模型1或模型2尚未加载，则给出明确提示，不进入推理
- 自动推理失败时，应保留已选择图片路径，方便用户修正后重试

### 6.4 手动执行推理

- 提供 `执行推理` 按钮
- 用户可在已经选择图片、并已加载模型后，手动再次执行一次推理
- 手动推理与自动推理共用同一套校验、后台线程、进度更新与结果刷新逻辑

### 6.5 获取模型信息

- 提供 `获取模型信息` 按钮
- 若仅加载模型1，则显示模型1信息
- 若仅加载模型2，则显示模型2信息
- 若模型1、模型2都已加载，则同时显示两份模型信息
- 若两个模型都未加载，则给出明确提示

### 6.6 释放模型

- 点击 `释放模型` 后，释放模型1、模型2实例
- 文本区输出 `模型已释放`
- 路径文本框不强制清空
- 若速度测试正在运行，应先停止速度测试再释放

### 6.7 速度测试

- 按钮为开关：未运行时显示 `速度测试`，运行中显示 `停止`
- 前置条件：模型1、模型2、图片路径都有效
- 启动时读取图片并缓存 RGB 图像，在内存中循环复用
- 运行中仅更新文本区统计，不刷新图像区
- 停止后恢复 UI 可用状态

## 7. 固定推理流程

### 7.1 第一步：模型1整图推理

推理入口先对整张 RGB 原图调用：

- `model1.Infer(fullImageRgb, params)`

其中：

- `params["with_mask"] = false`
- 输入图像固定为 RGB
- 模型1结果会先经过图像边界裁剪

### 7.2 第二步：提取中心点

中心点提取规则如下：

- 若结果为普通框：`center = (x + w / 2, y + h / 2)`
- 若结果为旋转框：`center = (cx, cy)`

### 7.3 第三步：在原图固定尺寸裁图

`buildCenteredCropContext()` 固定使用：

- `crop_width = 128`
- `crop_height = 192`

裁图规则如下：

- 以目标中心点为基准
- 在原图上请求一个固定大小窗口
- 若请求区域部分越界，则输出尺寸仍保持 `128 x 192`
- 越界区域以黑色补齐
- 同时记录该 crop 对应整图的平移量 `translateX / translateY`

### 7.4 第四步：模型2按 batch 推理

模型2 batch 推理实现要点如下：

- 通过 `getModelMaxBatchSize(model2)` 自动推断最大 batch
- 将所有 crop 按 batch 大小切分为多个 chunk
- 线程数由 `模型2线程数` 输入框给出，范围 `1~32`
- 实际并发线程数不会超过 chunk 数量
- 所有线程共享同一个 `model2` 实例执行 `InferBatch`
- 失败的 batch 会记日志，但不会中断其它 batch

### 7.5 第五步：模型2结果回写整图坐标

回写方式为纯平移：

- `full_x = local_x + translateX`
- `full_y = local_y + translateY`

随后结果还会再次执行图像边界裁剪，确保最终显示对象位于图像有效区域内。

### 7.6 第六步：组织最终结果

- 最终结果只收集模型2回写后的对象
- 模型1结果不进入最终显示
- 图像区只显示 `finalObjects`
- 文本区展示本次推理统计与日志

## 8. 结果展示规则

### 8.1 图像区

- 图像区始终显示原图
- 可视化对象只使用模型2最终结果
- 不显示模型1中间定位框
- 不显示中间裁图
- 支持缩放、拖拽等交互，沿用现有 `ImageViewerWidget`

### 8.2 文本区

单次推理完成后，文本区至少展示：

- 图片路径
- 模型1路径
- 模型2路径
- 固定裁图大小 `128 x 192`
- 模型1目标数
- 有效裁图数
- 模型2最大 batch
- 模型2线程数
- 最终结果数
- 推理耗时
- 运行日志

模型信息展示时，文本区支持展示：

- `模型1信息`
- `模型2信息`
- 模型原始 JSON 信息或 `model_info` 子对象

### 8.3 单次推理文本示例

```text

模型1: D:\models\model1.dvst
模型2: D:\models\model2.dvst
固定裁图大小: 128 x 192
模型1目标数: 42

模型2线程数: 4
```

## 9. 异常与边界处理

- 未加载模型1：提示“请先加载模型1！”
- 未加载模型2：提示“请先加载模型2！”
- 未选择图片：提示“请先选择图片文件！”
- 图片解码失败：报错“图像解码失败”
- 推理进行中再次点击：提示“已有推理任务在运行”
- 速度测试运行中尝试单次推理：提示需先停止速度测试
- 速度测试运行中尝试加载模型：提示需稍后再操作
- 模型1无结果：日志输出“模型1未检测到目标。”
- 某个裁图完全落到图外：跳过该目标并记录日志
- 模型2某个 batch 推理失败：记录日志，继续处理其它 batch
- 回写结果完全落到图外：丢弃
- 回写结果部分越界：裁剪后再显示
